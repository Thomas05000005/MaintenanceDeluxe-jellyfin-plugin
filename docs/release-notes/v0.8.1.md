# 🔴 MaintenanceDeluxe v0.8.1 — Hotfix critique Jellyfin 10.11.9

**Release patch critique.** Si tu tournes sur Jellyfin **10.11.9 ou plus récent**, tu dois upgrader. v0.8.0 crash le serveur de façon spectaculaire.

## Ce qui se passait avec v0.8.0 sur 10.11.9

```
System.MissingMethodException: Method not found:
'System.Collections.Generic.IEnumerable`1<...User>
 MediaBrowser.Controller.Library.IUserManager.get_Users()'
```

Cette exception remontait :
- À chaque appel à `GET /MaintenanceDeluxe/users-summary` (widget whitelist dans l'admin)
- À chaque appel à `GET /MaintenanceDeluxe/announcements/admin` (count total users)
- À l'activation de la maintenance (`MaintenanceHelper.ActivateAsync`)

**Mais surtout** : elle remontait aussi dans les handlers `ItemAdded`, `ItemUpdated`, `ItemRemoved` de Jellyfin via la stack DI partagée — cassant l'indexation de bibliothèque et plein d'autres fonctionnalités core.

## Pourquoi

Jellyfin a supprimé la propriété `IUserManager.Users` (qui retournait `IEnumerable<User>`) entre les versions **10.11.6** et **10.11.9**. Remplacée par la méthode `GetUsers()`. C'est un breaking change ABI introduit en patch — pas en minor — ce qui est exceptionnel. Notre plugin était build contre `Jellyfin.Controller 10.11.6`, donc le DLL référençait l'ancienne propriété, et le runtime Jellyfin 10.11.9 ne la trouvait plus.

## Le fix

Trois substitutions mécaniques :

```csharp
// AVANT (v0.8.0, casse sur 10.11.9)
_userManager.Users.Select(...)
_userManager.Users.Count()
userManager.Users.ToList()

// APRÈS (v0.8.1, marche sur 10.11.9)
_userManager.GetUsers().Select(...)
_userManager.GetUsers().Count()
userManager.GetUsers().ToList()
```

Plus :
- `Jellyfin.Controller` + `Jellyfin.Model` bumpés de `10.11.6` → `10.11.9` dans le csproj (plugin et tests)
- `targetAbi: 10.11.6.0` → `10.11.9.0` dans le manifest

## Compatibilité

| Jellyfin server | Plugin version compatible |
|---|---|
| 10.11.6 → 10.11.8 | **v0.8.0** (dernière) |
| 10.11.9+ | **v0.8.1** (cette release) |

Le manifest expose `targetAbi: 10.11.9.0`, ce qui veut dire que Jellyfin ≤ 10.11.8 refusera l'install de v0.8.1+ via le catalog. Si tu es bloqué sur une vieille version, reste sur v0.8.0. Pour tous les autres : upgrade.

## Tests

279/279 tests xUnit verts sur le nouveau SDK. Pas de test modifié — la logique métier n'a pas bougé, seule la signature de l'API Jellyfin.

## Aucune autre modif

C'est strictement un hotfix ABI. Aucun comportement utilisateur changé. Aucune migration de config. Tu peux upgrader v0.8.0 → v0.8.1 sans regarder.

## Note : autre plugin cassé sur 10.11.9

Tu as aussi rapporté que `Jellyfin.Plugin.JellyfinEnhanced.Services.WatchlistMonitor` plantait avec la même `MissingMethodException`. C'est exactement le même type de problème, mais sur un autre plugin (pas le nôtre). À signaler à son mainteneur — le fix est trivial (un `.GetUsers()` au lieu de `.Users`). Le plugin auteur peut se baser sur cette release notes pour comprendre.
