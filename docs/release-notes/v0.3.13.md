# 🚀 MaintenanceDeluxe v0.3.13 — Modal centrée, style premium, templates rapides

Trois fixes ciblés sur l'**expérience visuelle** et l'**ergonomie admin** du système d'annonces. Quand un user a créé sa première annonce, elle apparaissait en haut-gauche comme un toast cassé, sans le style premium attendu. Cette release force le centrage propre et ajoute des **modèles rapides** pour ne plus retaper la même structure à chaque annonce.

## ✨ Highlights

### 🪟 Modal enfin centrée plein écran

**Le bug** : la modal post-login apparaissait en haut-gauche à ~360 px de large, sans le fond noir dimmé, sans le styling premium. Visuel cassé, complètement à côté du design attendu.

**Cause technique** : Jellyfin a un ancestor (probablement `body` ou un wrapper interne) avec une propriété `transform` / `filter` / `will-change` / `contain` / `perspective` posée par le framework React. Ces propriétés créent un **nouveau containing block** pour `position:fixed`. Conséquence : notre overlay `position:fixed; inset:0` se re-anchorait à cet ancestor au lieu du viewport, et finissait dans une dimension étrange.

**Fix triple-defense** :
- **`!important` sur les positions critiques** dans le CSS injecté : `position:fixed !important; top:0 !important; left:0 !important; right:0 !important; bottom:0 !important;` + `width:100vw !important; height:100vh !important;`. Couvre tous les cas où un override CSS Jellyfin viendrait écraser nos propriétés
- **Append à `document.documentElement` (html), pas à `body`** : `<html>` n'a pas de transform / filter dans le DOM Jellyfin. La modal s'attache au plus haut niveau possible, à l'abri de tout containing block parent
- **Inline `overlay.style.cssText` avec `!important`** directement sur l'élément, en plus du CSS injecté. Les styles inline battent **toutes** les classes CSS, donc même si Jellyfin Media Player sert l'ancien `banner.js` depuis son cache Electron (qui n'avait pas `!important`), la modal s'affiche correctement. Belt-and-suspenders contre les caches agressifs
- **`box-sizing: border-box`** explicite pour que padding ne casse pas les 100vw/100vh
- **`margin: 0 !important`** pour neutraliser toute marge body héritée
- Opacity manipulée via `overlay.style.opacity` directement (pas via classe `.visible`) pour rester dans le même tier de spécificité que cssText et que la transition fonctionne quand même

### 🎨 Style premium retouché

- **`backdrop-filter: blur(8px)`** (était à `6px`) : effet de flou plus marqué sur le contenu Jellyfin derrière
- **Fond `rgba(0,0,0,0.68)`** (était à `0.62`) : plus de contraste, la card ressort mieux
- Le reste du style (palette velours/or, accent colorisé par importance, transitions, hover) inchangé — il était déjà bien, c'est juste qu'il n'était pas appliqué à cause du containing block

### 🪶 Modèles rapides d'annonces (templates)

Tu m'avais demandé "des préréglages pour que ce soit plus rapide" — voilà. 7 modèles pré-remplis disponibles en un click dans le tab Annonces :

| Modèle | Usage |
|---|---|
| ⚡ **Mise à jour serveur** | Update Jellyfin / plugin avec checklist de nouveautés (importance: update) |
| 🎬 **Nouveaux films** | Annonces hebdomadaires de contenus (ciblé non-admins) |
| 🔧 **Maintenance prévue** | Pre-warning avec date/heure (importance: warning) |
| 🚀 **Amélioration des perfs** | Pré-rempli avec **2 comparisons** "Latence streaming 200→140 ms · -30%" et "Démarrage 4s→2s · 2x" (importance: update) |
| 📢 **Annonce communautaire** | Format libre, info classique |
| 🎉 **Évènement / soirée** | Format dédié aux évènements ponctuels avec placeholder de date/heure |
| ⚠️ **Alerte critique** | Format dédié aux annonces importantes (importance: critical, accent rouge) |

**Comment ça marche** :
- Section "Modèles rapides" en haut de l'onglet Annonces avec **7 boutons inline** (style or sur fond ambre subtil)
- Click sur un modèle → une nouvelle annonce **pré-remplie** est ajoutée à la liste, automatiquement expandée pour édition
- Tu n'as plus qu'à modifier le contenu spécifique (titre custom, body précis, dates) au lieu de tout structurer

Le bouton historique "+ Nouvelle annonce" est renommé **"+ Annonce vide"** pour clarifier l'alternative (template vide pour les cas où tu veux partir from scratch).

## 🔧 Détails techniques

- `ANN_TEMPLATES` est un array constant dans `admin.js` — facile à étendre dans une future release (sondage user, "regarde aujourd'hui", "fin d'année", etc.)
- Chaque template définit : `icon`, `title`, `body` (markdown), `importance`, `comparisons[]`, `targetRoles[]`, `targetUserIds[]`, `ctaLabel/Url`. Le helper `addAnnouncementFromTemplate` clone proprement (deep copy des sous-arrays) pour ne pas partager les références
- L'ID est laissé vide (`""`) — le serveur en assignera un nouveau au prochain save
- Aucune modification du format `Announcement` stocké server-side — templates appliqués 100% côté client

## 🧪 Tests

- **121 tests xUnit** inchangés (le changement est CSS + UI admin, pas de logique business)
- **Validation manuelle** :
  1. Onglet *Annonces* → clique un modèle → vérifier qu'une nouvelle annonce apparaît pré-remplie
  2. Active-la, déconnecte-toi, reconnecte-toi → la modal pop **centrée plein écran** avec fond dimmé et style premium

## 📝 Notes

- Le containing block bug touche **toute modal `position:fixed` dans Jellyfin**. Le pattern `documentElement.appendChild + !important` est la solution propre. Si tu écris un autre plugin avec une modal, applique le même
- Si tu vois encore l'ancienne UI après upgrade : la fix ETag de v0.3.11 prend effet, mais sur la transition v0.3.12 → v0.3.13 un `Ctrl+Shift+R` reste utile **une seule fois** (le browser doit re-fetcher banner.js avec son nouveau ETag)
- Les 7 templates ne sont pas exhaustifs — ils couvrent les patterns les plus courants. Tu peux toujours créer une annonce blank et personnaliser totalement
