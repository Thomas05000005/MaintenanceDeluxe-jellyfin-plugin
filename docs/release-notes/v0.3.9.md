# 🚀 MaintenanceDeluxe v0.3.9 — Le système d'annonces post-login

Cette release introduit une **nouvelle famille de communication** entre l'admin et ses utilisateurs : la modal *What's New*. Elle s'ajoute aux deux dispositifs existants (overlay maintenance plein écran, bannières barre-haut) pour couvrir un troisième besoin — pousser un message **important et personnalisable** à chaque utilisateur **une seule fois**, juste après sa connexion.

Le markdown enrichi avec liens cliquables est livré dans la foulée, et profite à la fois aux annonces et aux release notes maintenance.

## ✨ Highlights

### 📣 Système d'annonces (What's New)

- 🎯 **Modal post-login dismissible** — apparaît centrée à la connexion, contrairement aux bannières (haut de page) et à l'overlay maintenance (bloquant).
- 👤 **Vu une seule fois par utilisateur** — tracking serveur (pas localStorage), persiste à travers tous les devices d'un même user.
- 🎚️ **Ciblage fin** : par rôle (`user` / `admin`) **ET** par UUID utilisateur spécifique. Les deux filtres sont combinés en ET logique — *"admin ET (Alice ou Bob)"*.
- 🎨 **4 niveaux d'importance** — `info` (bleu) / `update` (vert) / `warning` (orange) / `critical` (rouge), chacun avec son accent coloré distinct.
- 📊 **Comparaisons structurées avant/après** — lignes `(Label, Before, After, Highlight)` rendues comme une mini-table dans la modal. Idéal pour stats perf : *"Latence streaming · 200 ms → 140 ms · -30%"*.
- 🔗 **Bouton CTA optionnel** — label + URL safe-scheme, ouvre dans un nouvel onglet.
- 🔁 **Reset 'vue par'** — un bouton dans l'éditeur efface le tracking, tout le monde re-voit l'annonce au prochain login (utile si tu as corrigé une faute après publication).

### 🖥️ Modal responsive multi-format

- 📱 **Mobile portrait (≤ 600 px)** — pleine largeur, comparaisons en stack vertical, padding réduit.
- 📲 **Mobile paysage / tablet / desktop (600 – 1920 px)** — modal 700 px centrée, table 3 colonnes.
- 📺 **TV (≥ 1920 px)** — modal 900 px, font +25%, padding +50%, boutons agrandis pour 10-foot UI.
- 🛡️ **Suppression intelligente** : la modal annonce ne s'affiche **pas** si l'overlay maintenance est actif au même moment (évite le double-stack).

### 🔗 Markdown enrichi : liens cliquables

- ✏️ Support `[texte](url)` dans le markdown des annonces **ET** dans les release notes maintenance existantes — *"d'une pierre deux coups"*.
- 🛡️ Validation server-side : whitelist http(s) / relatif (rejette `javascript:`, `data:`, etc.).
- 🔐 Échappement HTML appliqué AVANT la transformation markdown — pas d'injection HTML possible via le body.

### 🧩 Admin UI : nouvel onglet *Annonces*

- 📋 Liste collapsible avec ratio « vu par X / Y utilisateurs » par annonce.
- ✍️ Éditeur complet : icon picker, titre, version, body markdown avec **aperçu live à côté**, importance segmented, checkbox active, ciblage rôles + multi-select utilisateurs spécifiques, comparaisons (add / remove rows), CTA.
- 💾 **Auto-save** à chaque modification de champ — pas besoin de cliquer Save manuellement pour ce tab.
- 🗑️ Boutons *Réinitialiser 'vue par'* et *Supprimer* par annonce.

## 📝 Notes techniques

- 🌐 5 nouveaux endpoints REST :
  - `GET /announcements/active` (any auth) — filtrées server-side per utilisateur
  - `POST /announcements/{id}/seen` (any auth) — mark seen, idempotent
  - `GET /announcements/admin` (admin) — full list + seen-counts
  - `POST /announcements/admin` (admin) — save list + multiMode
  - `POST /announcements/admin/{id}/reset-seen` (admin) — reset tracking
- 🛡️ DTO retourné aux utilisateurs **strippé** de `targetRoles` et `targetUserIds` — empêche un utilisateur lambda d'inférer qui d'autre a été ciblé par la même annonce.
- 🧪 **`AnnouncementHelper`** : 9 fonctions pures testables sans Plugin.Instance (IsTargetedAtUser, HasUserSeen, SelectDeliverableForUser, MarkSeen, ResetSeen, PruneOrphanedSeenEntries, NormaliseImportance, NormaliseMultiMode, NormaliseTargetRoles, NormaliseTargetUserIds).
- 🛡️ Test anti-drift `IntentionallyAdminOnly` renforcé : la whitelist explicite des champs admin-only force une revue de sécurité consciente à chaque nouveau champ ajouté à PluginConfiguration.

## 🚧 Limites Phase 1

Cette release implémente le mode `one-at-a-time` côté client. Les modes `carousel` (navigation flèches) et `stack` (toutes empilées scrollable) sont stockés côté serveur mais retombent sur `one-at-a-time` à l'affichage. Phase 2 prévue.

Le viewport simulator admin (presets mobile/tablet/TV/desktop) et les features additionnelles (image optionnelle, schedule fixed/annual/weekly/daily, brouillon/publié, auto-expire) viendront aussi en Phase 2.
