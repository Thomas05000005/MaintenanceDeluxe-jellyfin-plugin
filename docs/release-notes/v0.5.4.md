# 🪟 MaintenanceDeluxe v0.5.4 — Viewport simulator admin

Tu veux savoir comment l'overlay maintenance s'affiche sur un téléphone, une TV 4K, ou une tablette en paysage ? Plus besoin d'ouvrir DevTools ou de changer de device. v0.5.4 ajoute une **toolbar de presets viewport** au-dessus de l'aperçu live dans la section Apparence.

## 🪟 Six presets

```
┌─────────────────────────────────────────────────────────────────────────┐
│ [Auto] [📱 Portrait] [📱 Paysage] [📋 Tablet] [🖥️ Desktop] [📺 TV]  1920×1080 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   [   APERÇU LIVE DE L'OVERLAY DANS LA TAILLE CHOISIE   ]              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

| Preset | Dimensions | Cible |
|---|---|---|
| **Auto** *(défaut)* | 100% du panneau | Comportement v0.5.3 — responsive selon ton écran admin |
| **📱 Portrait** | 375 × 667 | iPhone récent en portrait, mobile entry-level |
| **📱 Paysage** | 667 × 375 | Mobile en paysage (rare mais utile pour valider) |
| **📋 Tablet** | 768 × 1024 | iPad ou tablette Android |
| **🖥️ Desktop** | 1366 × 768 | Laptop standard, écran 1080p en zoom |
| **📺 TV** | 1920 × 1080 | HDTV, Jellyfin sur Android TV / Apple TV |

## 💾 Persistance

Ton choix de viewport est **stocké en localStorage** sous la clé `jf-md-vp-preset`. Donc si tu fais Tablet → refresh → tu retombes sur Tablet. Permet de garder ton viewport préféré pendant que tu itères sur l'apparence.

## 🔧 Sous le capot

Implémentation 100% client. Le backend C# n'est pas touché par cette release — c'est cosmétique, lié uniquement à l'interface d'administration.

**HTML** : nouvelle div `.jf-vp-toolbar` ajoutée au-dessus de l'iframe dans `configPage.html`. Le wrapper `.jf-vp-frame` enveloppe l'iframe.

**CSS** : 5 classes scopées dans `admin.css` :
```css
.jf-vp-frame.preset-mobile-portrait #appPreviewIframe {
  width: 375px !important; height: 667px !important; min-height: 0 !important;
}
```
Le `!important` est nécessaire pour battre les styles inline existants `width:100%;height:100%;min-height:540px` posés sur l'iframe.

**JS** : nouvelle fonction `initViewportSimulator()` appelée au boot dans `initLivePreview()`. Wire les boutons via event delegation, applique les classes, sauve/restaure localStorage. ~50 lignes.

**Wrapper scroll horizontal** : pour Desktop (1366px) et TV (1920px) qui dépassent la largeur du panneau admin, le wrapper `.jf-vp-frame` a `overflow: auto`. Tu peux scroller horizontalement pour voir l'overlay dans toute sa largeur, comme dans un vrai navigateur.

## 🎁 Pour toi

- Tu travailles sur l'apparence pour un user TV → clic **📺 TV**, ton aperçu passe instantanément en 1920×1080. Plus de mauvaises surprises au déploiement.

- Tu compares l'effet d'un changement de `Card opacity` à différentes tailles → bascule entre Mobile / Tablet / Desktop sans quitter la config.
