# 🎠 MaintenanceDeluxe v0.5.0 — Carousel + Stack modes

Phase 2 partielle des annonces. Depuis v0.3.9 (Phase 1), le sélecteur **"Mode d'affichage"** dans l'onglet Annonces proposait trois choix : `one-at-a-time`, `carousel`, `stack`. Mais le client n'avait implémenté que **`one-at-a-time`** — les deux autres étaient stockés serveur mais ignorés. Si tu choisissais carousel en admin, ça affichait quand même une modale séquentielle.

v0.5.0 ferme la promesse.

## 🎠 Mode `carousel`

Quand plusieurs annonces non-vues sont à délivrer, une **seule modale** avec deux chevrons gauche/droite et un **compteur "1 / N"** en bas.

```
┌──────────────────────────────────────┐
│                                      │
│  ‹       📣  Titre de l'annonce   ›  │
│            v1.2.3 · 18 mai 2026      │
│            Lorem ipsum...            │
│                                      │
│                         [Compris]    │
│                                      │
│              · 2 / 5 ·               │
└──────────────────────────────────────┘
```

- **Chaque slide a son propre thème.** Si l'annonce 1 est en Velours et l'annonce 2 est en Néon, l'overlay re-rend le thème complet quand tu passes de l'une à l'autre (palette + police + animation pulse pour Néon).
- **Marquage "vu" fire-and-forget** à chaque transition (POST `/announcements/{id}/seen`). Le close final marque aussi la slide visible. Donc si tu fais juste `←` puis ferme, les 2 slides sont marquées vues.
- **Navigation clavier** : `←` / `→` pour naviguer, `Escape` pour fermer, `Enter` sur "Compris" pour fermer.
- **Responsive** : sur écrans ≤ 700px, les chevrons collapsent en boutons flottants en bas (les flèches latérales prenaient trop de place sur mobile portrait).

## 📚 Mode `stack`

Toutes les annonces empilées verticalement dans une **seule modale scrollable**.

```
┌──────────────────────────────────────┐
│         3 ANNONCES                   │
├──────────────────────────────────────┤
│  📣  Titre annonce 1                 │
│      Lorem ipsum...                  │
├──────────────────────────────────────┤
│  ⚠️  Titre annonce 2                 │
│      Lorem ipsum...                  │
├──────────────────────────────────────┤
│  ✨  Titre annonce 3                 │
│      Lorem ipsum...                  │
├──────────────────────────────────────┤
│     [ Tout marquer comme vu ]        │  ← sticky
└──────────────────────────────────────┘
```

- **Header** : "N annonces" en haut, capitales fines.
- **Footer sticky** : un seul bouton "Tout marquer comme vu" en bas. Pas de bouton "Compris" par carte — c'est le bouton global qui POST `/seen` sur les N IDs en parallèle (fire-and-forget).
- **Toutes les cartes partagent le thème de la première.** Justification : mixer 4 thèmes empilés (Velours sombre + Néon pulsant + OLED noir + Glass flouté) dans la même modale serait visuellement épileptique. L'override par annonce reste respecté en mode `one-at-a-time` et `carousel` (où l'overlay re-rend entre slides), mais en stack on force le thème de la première pour la cohérence visuelle.
- **L'overlay scrolle** (pas chaque card individuellement) — donc le scroll est continu de haut en bas, pas de scroll-trap dans une carte.

## 🎯 Edge cases

- **Une seule annonce non-vue** → modale classique (pas de carousel-à-1-slide avec chevrons grisés ridicules, pas de stack-à-1-carte avec header "1 annonces" bizarre).
- **Zéro annonce non-vue** → rien ne s'affiche, comme avant.
- **Erreur réseau pendant le fetch** → silent, comme avant. Les annonces non-critiques ne doivent jamais bloquer l'expérience post-login.

## 🔧 Refactor sous le capot

Trois fonctions extraites pour permettre les 3 modes de partager le rendu :

- **`buildAnnouncementCardHtml(a, withOk)`** : fonction pure, retourne le HTML d'une carte annonce. Le flag `withOk` contrôle l'affichage du bouton "Compris" (vrai pour one-at-a-time / carousel, faux pour stack).
- **`createAnnouncementOverlay(themeKey, accent)`** : crée un overlay vide (positioning, backdrop, thème, accent var en CSS variable). Utilisé par les 3 modes.
- **`buildAnnouncementModal(a)`** : wrapper qui appelle les deux helpers. C'est l'API publique pour le mode one-at-a-time.

CSS : 26 nouvelles règles scopées via `#jf-ann-overlay.jf-ann-carousel` et `#jf-ann-overlay.jf-ann-stack`. Zéro impact sur le mode existant.

## 🎁 Pour toi

Côté admin, **rien ne change visuellement** dans l'éditeur. Le sélecteur de mode d'affichage qui existait déjà depuis v0.3.9 fait maintenant ce qu'il prétend.

Côté user final :
- Si tu publies **plusieurs annonces actives** en même temps et que le mode est `carousel`, ils auront une modale unique avec navigation flèches au lieu de devoir fermer N popups.
- Si le mode est `stack`, ils auront tout d'un coup dans une longue modale scrollable et un seul clic pour tout marquer vu.
- Si tu n'as qu'une annonce active à la fois (cas le plus courant), **tu ne vois aucune différence**.
