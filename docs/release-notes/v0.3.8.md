# 🧹 MaintenanceDeluxe v0.3.8 — Pre-flight, extraction CSS, audit fonctionnel

Cette release est le résultat direct d'un **audit fonctionnel complet** du plugin. Trois axes : une nouvelle feature qui aurait dû exister dès le début (pre-flight check), un nettoyage de code mort + bruit log, et une extraction CSS qui finit le travail de modularisation entamé en v0.3.7.

## ✨ Highlights

### 🛡️ Pre-flight check sessions actives

- 🎯 **Avant d'activer la maintenance**, l'admin voit un modal de confirmation listant les utilisateurs en train de streamer :
  ```
  Alice — Dune: Part Three (sur Chrome)
  Bob   — Some Like It Hot (sur Apple TV)
  
  Activer la maintenance va leur couper le flux.
  Confirmer ?
  ```
- 🎬 **Plus jamais de coupure en plein film** pour ta famille / tes amis. La feature qui aurait dû exister dès le début.
- 🌐 Nouvel endpoint `GET /MaintenanceDeluxe/active-sessions` (admin-only) qui retourne les sessions avec `NowPlayingItem` non-null.
- 🛡️ **Annulation propre** : si l'admin clique *Annuler* dans le modal, la case "Active" se remet automatiquement à son état serveur — aucune requête POST envoyée, pas d'incohérence UI.

### 🎨 Extraction CSS de la page admin

- ✂️ **359 lignes de CSS** extraites de `configPage.html` vers un nouveau fichier `Configuration/admin.css` (EmbeddedResource).
- 🌐 Nouvel endpoint public `GET /MaintenanceDeluxe/admin.css` servi avec `nosniff` + `Cache-Control: max-age=300`.
- 📉 `configPage.html` passe de **1329 à 971 lignes** (-27%). Cumul depuis v0.3.5 : **-78%** (4376 → 971).
- 🛡️ Nouveau garde-fou CI **anti-inline-style**, symétrique au guard anti-inline-script de v0.3.7. Toute régression d'inline CSS bloquée à la PR.

### 🧰 Factorisation BannerController

- 🔧 Les **3 endpoints d'assets** (`banner.js`, `admin.js`, `admin.css`) factorisés via un helper privé `ServeEmbeddedAsset(resourceName, contentType)`. Headers (`nosniff`, `Cache-Control`) posés à un seul endroit.
- ✅ Suggestion follow-up du code review v0.3.7 appliquée.

## 🧹 Nettoyages

- 🗑️ **`scripts/check_inline_script.py` supprimé** — mort code depuis v0.3.7 (le inline JS a été extrait dans admin.js, plus rien à parser).
- 🤫 **Préfixe `[MaintenanceDeluxe]` retiré** de tous les logs C# (5 fichiers, ~26 occurrences). Depuis v0.3.7 (`BeginScope`) et grâce au `LogCategory` automatique de `Microsoft.Extensions.Logging`, ce préfixe était du bruit pur. **Note** : banner.js conserve son préfixe dans `console.debug`/`console.warn` — utile côté DevTools pour identifier la source.

## 📚 Documentation

- 📖 Nouveau fichier `docs/banners.md` (~ 170 lignes) qui documente complètement le **système de bannières** rotation + permanent override + schedules 5 types + routes wildcards + color presets + dismiss cross-tab. Ce système était puissant mais sous-promu — il a maintenant sa doc dédiée.
- 📚 README : table des endpoints à jour (**12 routes**) + section *Banner system* qui pointe vers la nouvelle doc + section *Announcement modals* (préparation v0.3.9).

## 📝 Notes techniques

- 🔍 L'audit JS Injector confirme que la **reflection est l'API publique officielle** du plugin tiers (commentaire explicite dans leur `PluginInterface.cs` : *"This method is designed to be called by other plugins via reflection"*). Le pattern actuel est volontaire, pas un workaround.
- 🧪 Aucun nouveau test xUnit dans cette release (les changes sont UI / headers / logs / docs / refacto — couverts par les checks existants).
