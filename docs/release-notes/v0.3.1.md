# 🔔 MaintenanceDeluxe v0.3.1 — Webhooks Discord/Slack/Generic + liste blanche utilisateurs

Cette release ouvre le plugin sur l'extérieur (notifications webhook) et permet de garder certains utilisateurs actifs pendant la maintenance.

## ✨ Highlights

### 🔔 Notifications webhook

- 📡 Envoi automatique d'un payload HTTP à chaque transition de maintenance (activation, désactivation).
- 🎯 Format auto-détecté depuis l'URL :
  - `discord.com/api/webhooks/...` → Discord embeds (couleurs + fields)
  - `hooks.slack.com/services/...` → Slack Block Kit
  - autre → JSON générique (Home Assistant, n8n, ntfy, IFTTT…)
- 🧪 Bouton **Tester la notification** avec rate-limit 1 appel / 5 s par IP.
- 🔒 **HTTPS uniquement** — les tokens webhook ne doivent jamais transiter en clair.
- 🛡️ Si le webhook est down, la maintenance s'active/se désactive quand même. Timeout 5s + 1 retry sur 5xx/timeout.
- 📚 Doc des payloads : `docs/webhooks.md`.

### 👥 Liste blanche utilisateurs

- ✅ Nouveau widget multi-select dans le tab Maintenance.
- 🎯 Les utilisateurs cochés conservent leur accès pendant la maintenance — parfait pour la famille, l'astreinte, un compte de test.
- 👻 Gestion des "ghost IDs" : les utilisateurs supprimés apparaissent en italique avec un bouton Retirer.
- ⚠️ Edge case : un user ajouté APRÈS l'activation reste désactivé jusqu'au prochain toggle.

### 🔐 Privacy

- 🛡️ L'endpoint public `GET /maintenance` ne renvoie plus les listes UUID ni l'URL webhook.
- 🎯 Nouveau DTO `PublicMaintenanceSnapshot` qui projette uniquement les champs nécessaires à l'overlay client.
