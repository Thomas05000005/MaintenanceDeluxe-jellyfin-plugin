# 🎨 MaintenanceDeluxe v0.4.0 — Thèmes visuels pour les annonces

Première release **mineure** (0.4.x) qui élargit le scope du plugin : jusqu'ici la modal d'annonces post-login n'avait **qu'un seul look** (velours-et-or, héritage du maintenance overlay). À partir de v0.4.0, **4 thèmes** sont disponibles, chacun avec sa propre palette, sa propre police et sa propre animation d'entrée. Tu choisis un thème par **défaut global**, et chaque annonce peut **surcharger** ce choix individuellement.

## ✨ Highlights

### 🪟 4 thèmes prêts à l'emploi

| Thème | Quand l'utiliser |
|---|---|
| 🟫 **Velours & Or** *(défaut)* | Le look actuel, raffiné, palette sombre + accent or. Pour les annonces classiques où tu veux rester cohérent avec le maintenance overlay. |
| ⚫ **OLED Total Black** | Fond `#000` pur, monochrome, police monospace JetBrains Mono. Idéal écrans OLED, look minimaliste sec, tech. |
| 💫 **Glow Néon (sobre)** | Halo coloré animé en pulse 4.5s (couleur dictée par l'importance — bleu / vert / orange / rouge), Space Grotesk géométrique futuriste, text-shadow lumineux sur le titre. |
| 🔮 **Glassmorphism** | Fond très flouté (28px blur saturate 1.5x) + carte semi-transparente avec bordure lumineuse, Manrope rond élégant. Look macOS / Apple Vision Pro. |

### 🔤 Polices web embarquées (140 KB total)

Plutôt que de dépendre de Google Fonts (CDN externe + tracking + risque CSP / firewall des reverse-proxies), les 4 polices sont **bundlées dans le plugin** :

- **Inter** Variable (48 KB) — sans-serif premium SaaS, utilisé par Linear / Vercel / Stripe
- **JetBrains Mono** Variable (40 KB) — monospace tech, lisibilité parfaite
- **Space Grotesk** Variable (22 KB) — sans-serif géométrique, look futuriste léger
- **Manrope** Variable (25 KB) — sans-serif rond, ergonomie type Apple

Servies par `GET /MaintenanceDeluxe/fonts/{slug}.woff2` avec `Cache-Control: public, max-age=31536000, immutable` — un seul téléchargement par navigateur, jamais réinvalidé tant que le fichier n'est pas renommé.

**Chargement lazy** côté banner.js : les `@font-face` sont injectées au premier affichage d'un thème custom, et le browser ne télécharge la `.woff2` que pour la `font-family` réellement utilisée par le thème actif (via `font-display: swap`). Donc si tu restes sur Velours, **rien ne se télécharge**.

### 🎨 Sélecteur global + override par annonce

L'onglet **Annonces** a maintenant **deux niveaux** de configuration thème :

1. **Thème visuel par défaut** *(haut de l'onglet)* : 4 boutons segmented. Le choix s'applique à toutes les annonces qui ne définissent pas leur propre thème. Sauvegarde automatique au changement (radio change → POST).

2. **Thème pour cette annonce** *(dans chaque éditeur)* : un select avec 5 options — `Hériter du défaut global` (par défaut) + les 4 thèmes nommés. Permet de mixer plusieurs styles dans la même liste (par ex. critical-warning en Néon rouge pulsant pour bien attirer l'œil, annonces classiques en Velours).

### 👁️ Aperçu live avec thème réel

La preview admin (panneau droit de l'éditeur d'annonce) **applique le thème** en temps réel : palette + police + animation (la pulse Néon visible aussi en aperçu).

Un petit **badge** en haut du preview affiche le thème actif :
- `VELOURS & OR` *(sans étoile)* → annonce hérite du défaut global
- `OLED TOTAL BLACK *` *(étoile)* → annonce surcharge le défaut

Tooltip sur le badge clarifie : "Hérité du défaut global" / "Surchargé pour cette annonce".

## 🔧 Détails techniques

### Côté serveur

- **`PluginConfiguration.AnnouncementTheme`** : nouveau champ string (défaut `"velours"`). Reflété dans `BannerClientConfig` (anti-drift test mis à jour automatiquement via le whitelist `IntentionallyAdminOnly`).
- **`Announcement.Theme`** : nullable string, `null` = hérite. Whitelist : `{velours, oled, neon, glass}`. Si une valeur unknown arrive, le `NormaliseAnnouncementThemeOverride` la drop (retourne `null` → inherit), pas de fallback violent qui écrase le choix global.
- **`NormaliseAnnouncementTheme(value)`** : pour le default global, fallback `"velours"` si unknown.
- **`NormaliseAnnouncementThemeOverride(value)`** : pour les overrides par annonce, fallback `null` si unknown (laisse hériter).
- **`GET /announcements/active`** : résout côté serveur le thème effectif pour chaque annonce (`override ?? global`) avant d'envoyer au client. Le client n'a pas à connaître le default global.
- **`GET /announcements/admin`** : retourne désormais le `theme` global en plus du `multiMode` pour pré-remplir le sélecteur.
- **`GET /fonts/{slug}.woff2`** : nouveau endpoint public, mappe le slug (4 valeurs autorisées) au resource embed correspondant. Path traversal impossible — la table de routage est un `Dictionary<string, string>` figé en static readonly.

### Côté client (banner.js)

- **`ANN_THEMES`** : objet qui mappe chaque key → `{ font, css }`. Ajouter un nouveau thème = ajouter une entrée + une option dans l'admin select + l'ajouter à la whitelist serveur. Aucune autre modification nécessaire.
- **`injectAnnTheme(themeKey)`** : injecte le CSS du thème **une seule fois** par DOM. Plusieurs thèmes peuvent coexister sur la même page (utile pour l'aperçu admin qui peut basculer entre thèmes).
- **`injectAnnFontFaces()`** : injecte les 4 `@font-face` aussi en lazy (au premier theme custom). `font-display: swap` + variable woff2 → texte visible immédiat, mise à jour du glyph quand la font streame.
- **Scoping CSS** : chaque thème ajoute la classe `.jf-ann-theme-<key>` sur l'overlay. Toutes les règles thème sont scopées via `#jf-ann-overlay.jf-ann-theme-X .jf-ann-modal {...}` pour ne JAMAIS leak hors de la modal.

### Côté admin UI

- `_annData.theme` (nouveau) : trace le default global côté client pour résolution preview.
- `ANN_THEME_OPTIONS` + `ANN_THEME_LABELS` (constants partagés) : référence unique des keys et libellés affichés.
- Auto-save sur changement du radio global, avec re-render immédiat de **toutes** les previews (via dispatch `input` event sur chaque row) avant la requête réseau pour feedback instantané.

## 🧪 Tests

**136 tests xUnit** (+15 cette release) — 100% verts :

- `NormaliseAnnouncementTheme_AcceptsFourThemesElseVelours` (7 cases) : valide la whitelist + fallback default
- `NormaliseAnnouncementThemeOverride_AcceptsFourThemesElseNull` (8 cases) : valide la sémantique "unknown → null" pour les overrides

L'anti-drift `DtoMirrorsAllNonMaintenanceJsonPropertiesOfPluginConfiguration` continue de passer — le nouveau `AnnouncementTheme` est exposé dans `BannerClientConfig` automatiquement.

### Validation manuelle

1. Onglet *Annonces* → changer le défaut global *Velours → OLED* → vérifier que toutes les previews passent en monospace fond noir, sauf celles avec override
2. Dans l'éditeur d'une annonce → changer son thème → seule cette preview change, le badge passe à `OLED TOTAL BLACK *`
3. Active une annonce, déconnecte / reconnecte → la modal réelle apparaît avec le thème choisi (palette + police + animation)
4. Sur écran OLED : tester le thème OLED en condition réelle — vrai noir pur, économie d'énergie maximale
5. Sur grosse résolution (1920+) : tester le Glow Néon pour voir la pulse en pleine taille

## 📝 Notes

- **Phase 1 délibérément limitée à 4 thèmes** pour rester maintenable. La Phase 2 (planifiée) ajoutera un **éditeur de thème custom** comme la fenêtre de maintenance, pour créer ses propres palettes / animations sans modifier le code.
- Si tu vois encore l'ancienne UI après upgrade : le cache busting v0.3.11 fait son boulot, mais sur la transition v0.3.13 → v0.4.0 un `Ctrl+Shift+R` une seule fois peut aider sur Jellyfin Media Player (Electron cache plus agressif).
- Les polices restent disponibles même pour des futurs usages : tout autre composant du plugin peut désormais référencer `font-family: 'JFAnnInter'` / `'JFAnnJetBrains'` / `'JFAnnSpaceGrotesk'` / `'JFAnnManrope'` sans recharger.
- L'animation `jfAnnGlowPulse` est ciblée 4.5s en `ease-in-out` — assez lente pour ne pas distraire, assez visible pour signaler "annonce importante". Si tu trouves trop, le thème Velours reste neutre.
