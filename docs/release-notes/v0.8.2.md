# 🔄 MaintenanceDeluxe v0.8.2 — Compat refresh Jellyfin 10.11.10

**Tu peux upgrader sans rien lire.** Pas de fix critique, pas de feature, pas de migration. Juste un rebuild contre le SDK Jellyfin 10.11.10 (sorti côté serveur) + un bump du `targetAbi` du manifest pour que Jellyfin 10.11.10+ accepte l'install sans warning.

## Pourquoi cette release

Jellyfin 10.11.10 est sorti côté serveur. v0.8.1 (build contre 10.11.9) marche déjà sur 10.11.10 grâce à la forward-compat Jellyfin same-minor — mais le manifest dit `targetAbi: 10.11.9.0`. Certains setups Jellyfin sont stricts là-dessus, donc je bump pour propreté.

## Audit API

J'ai diffé les XML docs des SDK 10.11.9 et 10.11.10 sur les interfaces qu'on utilise :

```
diff <(grep IUserManager 10.11.9/MediaBrowser.Controller.xml) \
     <(grep IUserManager 10.11.10/MediaBrowser.Controller.xml)
# → (empty, no difference)

diff <(grep ISessionManager 10.11.9/MediaBrowser.Controller.xml) \
     <(grep ISessionManager 10.11.10/MediaBrowser.Controller.xml)
# → (empty, no difference)
```

**Zéro changement** sur :
- `IUserManager.GetUsers()`
- `IUserManager.GetUserById(Guid)`
- `IUserManager.GetUserByName(string)`
- `IUserManager.GetUserDto(User, string)`
- `IUserManager.UpdatePolicyAsync(Guid, UserPolicy)`
- `ISessionManager.Sessions`

Contrairement au saut **10.11.6 → 10.11.9** qui avait supprimé `IUserManager.Users` (cause de v0.8.1), le saut **10.11.9 → 10.11.10** ne casse rien.

## Compatibilité finale

| Jellyfin server | Plugin recommandé |
|---|---|
| 10.11.6 → 10.11.8 | **v0.8.0** (dernier compatible) |
| 10.11.9 | v0.8.1 ou v0.8.2 (les deux marchent) |
| **10.11.10+** | **v0.8.2** ✨ |
