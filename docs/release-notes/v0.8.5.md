# 🩹 MaintenanceDeluxe v0.8.5 — Corrections UX & accessibilité

**Recommandée.** Pas de nouvelle fonctionnalité, pas de migration : c'est une release de **corrections**, issues d'un 2ᵉ audit ultra-complet de toutes les couches (UX, design, performance, code, accessibilité, docs). L'audit n'a trouvé **aucun problème critique** ; cette version corrige les 6 plus importants (+ quelques bonus). Compatible **Jellyfin 10.11.10 et 10.11.11**.

## Ce qui change pour toi

- **Les modèles rapides d'annonce marchent enfin comme annoncé.** Cliquer « 🎃 Halloween (annuel) » ou « 🎄 Noël (annuel) » créait une annonce **sans planning** — donc affichée toute l'année. Désormais le planning, l'auto-expiration, le thème et l'image du modèle sont bien repris.
- **L'écran de maintenance est accessible au clavier / lecteur d'écran.** Il a maintenant la même qualité que les fenêtres d'annonce : le focus reste piégé dedans, la page derrière ne défile plus, et le focus revient à sa place à la fermeture.
- **Le mode « carousel » d'annonces ne se répète plus.** Avant, si tu cliquais « Compris » sans faire défiler toutes les cartes, le carousel revenait à chaque connexion. Toutes les cartes sont maintenant marquées comme vues à la fermeture.
- **Un redémarrage programmé à la fin d'une maintenance se déclenche vraiment.** Il était annulé silencieusement quand il tombait pile à la fin de la fenêtre.
- **Robustesse** : le nombre d'annonces / messages / presets est plafonné (anti-gonflement du fichier de config), et une image d'annonce dont l'URL est morte se cache au lieu d'afficher une icône cassée.
- **Outillage** : `make bump` (interne) réparé, et le README annonce désormais la bonne version Jellyfin (10.11.10).

## Qualité

- **366 tests** xUnit verts (+3 pour les nouveaux plafonds). Build sans warning, script client ASCII-safe.
- Le correctif a été repassé par une **revue adversariale multi-agents** : aucune régression critique ou élevée (une seule broutille corrigée dans la foulée — le nettoyage a11y de l'écran « de retour en ligne »).
- **Compatibilité re-vérifiée** : build + tous les tests contre le SDK Jellyfin 10.11.11, plus un smoke test sur de vrais serveurs 10.11.10 **et** 10.11.11.
