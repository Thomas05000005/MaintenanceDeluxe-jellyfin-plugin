# ♿ MaintenanceDeluxe v0.8.0 — A11y + UX + CI robustesse

**Tour de fixes complet** : 16 améliorations appliquées par **4 agents en parallèle** sur des fichiers distincts. +13 tests xUnit (266 → 279). Aucun nouveau champ — uniquement défensif.

## ♿ Accessibilité (8 fixes)

### Focus trap dans toutes les modales

Avant v0.8.0 : `Tab` depuis la dernière modale sortait vers le contenu de fond. Maintenant, Tab et Shift+Tab cyclent dans la modale :

- **`dangerConfirm`** (admin) — Tab : Cancel ↔ Danger
- **`showAnnouncementModal`** (banner.js) — focus sur le bouton OK initial, Tab cycle sur les liens CTA + OK
- **Mode `carousel`** — focusables = [prev, next, ok] en skippant les disabled
- **Mode `stack`** — focus sur "Tout marquer comme vu" + cycle sur les CTA des cartes

Le focus est capturé via `document.activeElement` avant l'ouverture et restauré au close. ESC ferme, Enter confirme (si non-bloqué).

### Scroll lock body

Quand l'overlay annonce s'ouvre, `documentElement.style.overflow = "hidden"` (+ `body.style.overflow = "hidden"` pour safety). Le scroll du contenu derrière la modale est verrouillé. Restore au close.

### Compteur carousel annoncé au screen reader

Avant : `overlay.innerHTML = html` recréait le `<div class="jf-ann-counter">` à chaque navigation → le screen reader ratait les changements (élément considéré "nouveau").

Maintenant : le compteur est créé **une fois** hors de `render()`, avec `aria-atomic="true"`. Seul `counterEl.textContent` mute à chaque navigation. Le screen reader annonce correctement "2 sur 3" quand l'admin appuie sur les flèches.

### prefers-reduced-motion

Quatre animations respectent maintenant la préférence système :

- Thème néon pulse 4.5s (banner annonce)
- Badge "dirty" pulse orange (admin)
- `jfDconfFade` overlay animation (dangerConfirm)
- Preview néon glow (admin preview)

Bloc dédié `@media (prefers-reduced-motion: reduce)` qui force `animation: none !important`.

### Contraste WCAG AA sur badges

Les pilules d'état admin avaient des ratios de contraste < 4.5:1 sur fond sombre. Corrigé :

| Badge | Avant | Après |
|---|---|---|
| `.draft` | `#E8A33D` + bg 0.18 | `#FFD27A` + bg 0.28 |
| `.expired` | `#D9534F` + bg 0.15 | `#FF8A87` + bg 0.30 |
| `.expiring` | `#E8A33D` + bg 0.12 | `#FFD27A` + bg 0.22 |
| `.schedule-incomplete` | `#D9534F` + bg 0.16 | `#FF8A87` + bg 0.30 |

### Touch targets 44×44 (WCAG 2.5.5)

- **`.jf-dconf-btn`** (admin modale) : `padding: 8px` → `12px 20px` + `min-height: 44px; min-width: 44px;`
- **`.jf-url-btn`** (banner popup URL) : `padding: 7px` → `12px 18px` + `min-height: 44px` + `inline-flex` pour centrage vertical

### Focus visible outline

Nouveau sélecteur `.jf-ann-field input:focus-visible, .jf-ann-field textarea:focus-visible` avec `outline: 2px solid rgba(201, 169, 110, 0.7); outline-offset: 1px;`. Les utilisateurs au clavier voient désormais clairement quel input est focalisé.

### `backdrop-filter` fallback solide

Pour les vieux Chromium TV (Tizen 4-5, webOS 3-4) qui n'ont pas `backdrop-filter`, un `@supports not (backdrop-filter)` force un fallback opaque sur `.jf-app-expanded` et `.theme-glass`. Plus de fond invisible/blanc sur ces devices.

## 🔧 UX (5 fixes)

### Webhook test button race condition

Avant : double-clic admin = 2 requêtes en vol simultanément. Maintenant : bouton désactivé avec texte "Test en cours…" pendant la requête, `reenable()` partagé entre `.then()` et `.catch()`. Le rate-limit serveur n'est plus mis à contribution pour rien.

### Import config Cancel cleanup

Bug : `_pendingImport` n'était pas reset sur Cancel → re-clic Apply sans nouveau fichier = ghost import.

Fix :
- `closeImportModal()` centralisée qui reset `_pendingImport = null` et désactive le bouton Apply
- Appelée sur Cancel / ESC / backdrop click
- `applyImportedConfig` garde-fou : si `_pendingImport === null`, affiche feedback orange "Sélectionne d'abord un fichier…"

### Release notes delete avec confirmation

Avant : `row.remove()` direct → admin perd 10 min de rédaction d'un clic. Maintenant : `dangerConfirm` avec délai 2s si la section a du contenu. Slot complètement vide = delete direct (pas de friction inutile).

### Draft autosave feedback

Avant : quota localStorage dépassé = `catch (e) {}` silencieux, admin perd ses brouillons sans le savoir.

Maintenant : flag `_draftSaveFailed` + `console.warn` one-shot. L'admin peut voir la cause dans la console s'il s'inquiète.

### datetime-local fallback

Les 3 inputs `type="datetime-local"` (`scheduledStart`, `scheduledEnd`, `scheduledRestart`) ont maintenant `placeholder="YYYY-MM-DDTHH:MM"` + `pattern="..."`. Les vieux Safari mobile (qui n'ont pas le picker natif) voient le format attendu.

## 🛠️ Infrastructure CI (3 fixes)

### `escape_non_ascii.py` BOM support

`encoding="utf-8"` → `encoding="utf-8-sig"`. Le BOM UTF-8 (parfois ajouté par certains éditeurs) est maintenant strippé automatiquement, plus de faux positifs ou crashes.

### `check_version_coherence.py` robustesse

`manifest[0]["versions"][0]["version"]` était sans try/except → crash brut sur manifest malformé. Maintenant : try/except sur `JSONDecodeError`, `KeyError`, `IndexError`, etc. avec message clair et exit code 2.

### `WebhookNotifier` méthodes accessibles aux tests

`SanitiseExceptionMessage` et `TryGetRetryAfterSeconds` passées `private static` → `internal static`. `InternalsVisibleTo` déjà configuré dans le csproj.

## 🧪 Tests (+13 cas)

### `TryGetRetryAfterSeconds_*` (4 Facts)

- `DeltaCapped60s` — 120s passé → retour ≤ 60

### `IsScheduleActive_Daily_MidnightWrap_ExactBoundaries`
Couvre les boundaries exactes 22:59 (exclu), 23:00 (inclus), 00:00 (inclus), 00:01 (inclus), 01:01 (exclu).
