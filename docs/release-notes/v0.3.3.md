# 🔐 MaintenanceDeluxe v0.3.3 — Audit complet : 12 correctifs (2 sécurité + 8 robustesse + 2 polish)

Cette release est l'aboutissement d'un **audit complet du plugin** sur 3 dimensions (qualité code, sécurité, alignement fonctionnel). Deux failles de sécurité fermées, huit issues de robustesse résolues, deux polish UX, et 13 nouveaux tests xUnit pour cimenter les invariants.

## 🚨 Sécurité critique

### 🔒 GET /config ne fuit plus les secrets

- 🛑 **Avant** : `GET /config` était `[Authorize]` (any auth) et retournait `PluginConfiguration` **entière**, incluant `Webhook.Url` (= secret Discord/Slack — quiconque le possède peut spam le canal) et les listes d'UUIDs utilisateurs.
- ✅ **Après** : retourne désormais un DTO scrubbé `BannerClientConfig` qui mirroir `PluginConfiguration` **sauf `MaintenanceMode`**. Nouvel endpoint `GET /config-admin` (`RequiresElevation`) pour la page admin qui a besoin du full state.
- 🧪 Test anti-drift `DtoMirrorsAllNonMaintenanceJsonPropertiesOfPluginConfiguration` qui garantit qu'aucun champ banner-facing n'est oublié dans le DTO.

### 🔒 Drift check périodique pendant la maintenance

- 🛑 **Avant** : `EnsureUsersDisabledAsync` ne s'exécutait **qu'une fois par lifetime** du process (au startup). Si un autre admin (ou la femme de l'admin) re-enable un utilisateur pendant la maintenance via la dashboard Jellyfin, le plugin ne le voyait plus jamais jusqu'au prochain redémarrage serveur.
- ✅ **Après** : `EnsureUsersDisabledAsync` est appelée à **chaque tick (1 min)** du scheduled task quand maintenance active. Si dérive, l'utilisateur est ré-disabled dans la minute.

## ✨ Robustesse (8 fixes)

- 🔧 **BUG-3+4 Rate-limit `/test-webhook`** : remplacé `ConcurrentDictionary<string, DateTime>` per-IP par un `long` global atomique (`Interlocked.CompareExchange`). Plus de memory leak, plus de bypass derrière reverse proxy.
- ⚡ **BUG-5** : `Cache-Control: public, max-age=10` sur `GET /maintenance` (banner.js poll à chaque nav SPA).
- 🔁 **BUG-7+8** : JS Injector retry exponentiel `5/15/45/120/300/600 s` (~18 min) au lieu de `3 × 5 s`. Nouveau helper `ScheduleRetry` avec try/catch propre dans le Task.Run.
- 🎯 **BUG-9** : `MaintenanceScheduleTask.ExecuteAsync` lit `Plugin.Instance` une seule fois par tick (cohérence inter-branches).
- 💾 **BUG-10** : `SaveMaintenance` plus de double `SaveConfiguration()` autour des helpers. Fenêtre de race fermée où un `GET /config-admin` concurrent voyait un état intermédiaire.
- 🌐 **BUG-6** : `Webhook URL` log warning si l'hôte n'est ni Discord ni Slack (generic webhooks toujours acceptés, juste un signal côté admin).
- 📏 **BUG-15** : Borne supérieure de **30 jours** sur `scheduledRestart`. Rejet avec 400 si l'admin saisit une date > now+30j (typos et configs polluées détectables).
- 🔔 **BUG-16** : Nouvel événement webhook `Restarting` envoyé (et awaité, avec timeout 5s) juste avant `_systemManager.Restart()`. Nouveau champ `WebhookSettings.NotifyOnRestart` (default `true`).

## ✨ Polish (2 fixes)

- 🔒 **BUG-11** : `escHtml` côté admin échappe désormais aussi l'apostrophe (`'` → `&#39;`). Latent — pas d'XSS aujourd'hui, mais durci pour les futurs ajouts.
- 🐛 **BUG-14** : `console.debug` dans banner.js guardé derrière `?md-debug=1`. Plus de chatter dans les DevTools de tes utilisateurs.
