# 🧪 MaintenanceDeluxe v0.3.5 — Tests d'intégration + bump GHA actions

Refactor qualité interne sans changement de comportement runtime. Couvre les invariants critiques de la logique business par tests + nettoie la dette CI/CD avant les deadlines de janvier 2026.

## ✨ Highlights

### 🧪 Couverture tests étendue

- 🔧 **4 fonctions pures extraites** dans `MaintenanceHelper` (`IsAdmin`, `IsDisabled`, `SelectUsersToDisable`, `SelectPreDisabledIds`) en `internal static` — la logique de partition utilisateurs (admin / déjà disabled / whitelisté / à désactiver) est désormais testable sans Plugin singleton ni mock IUserManager.
- 🧪 **9 nouveaux tests xUnit** dans `MaintenanceHelperTests` :
  - Sélection cible enabled non-admin hors whitelist
  - Whitelist vide, all-users-are-admins, no-users edge cases
  - Pre-disabled UUIDs distincts des admins déjà disabled
  - **Invariant critique** : `toDisable` et `preDisabled` sont **toujours disjoints** (sinon `DeactivateAsync` ré-activerait à tort)
  - Whitelist > enabled status
- ✅ **83 tests au total** (74 v0.3.4 + 9 nouveaux), tous verts.

### ⚙️ GitHub Actions à jour

- 🔄 Bump des 4 actions aux dernières majeures :
  - `actions/checkout@v4` → `@v6`
  - `actions/setup-dotnet@v4` → `@v5`
  - `actions/setup-node@v4` → `@v6`

## 📝 Notes techniques

- ✅ **Aucun changement de comportement observable runtime**. `ActivateAsync` réécrit pour utiliser les helpers extraits, mais les comportements identiques bit-pour-bit.
