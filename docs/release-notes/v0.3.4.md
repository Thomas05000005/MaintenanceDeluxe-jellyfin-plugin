# ✨ MaintenanceDeluxe v0.3.4 — Polish post-audit (UX webhook + headers + docs)

Suite directe à v0.3.3 (audit complet 12 fixes). Cette release applique les retouches identifiées comme "à faire en priorité" lors du post-mortem : surface UI manquante, durcissement HTTP léger, et préparation aux deadlines GitHub Actions.

## ✨ Highlights

### 🔔 Surface UX : checkbox NotifyOnRestart

- ☑️ Nouvelle checkbox **"Notifier avant un redémarrage serveur"** dans le tab Maintenance, section *Notifications*.
- 🎯 Pilote le champ `WebhookSettings.NotifyOnRestart` ajouté server-side en v0.3.3 mais qui n'avait pas encore de surface admin. Default `true`, rétro-compatible avec les anciennes configs.

### 🛡️ Durcissement headers HTTP

- 🔒 `GET /MaintenanceDeluxe/banner.js` → `X-Content-Type-Options: nosniff` + `Cache-Control: public, max-age=300` (le script est immutable pour la lifetime d'un plugin version → 5 min de cache économise des fetchs sans masquer les updates).
- 🔒 `GET /MaintenanceDeluxe/preview.html` → `nosniff` + `X-Frame-Options: SAMEORIGIN` (anti-clickjacking ; l'iframe legitimate est servie par Jellyfin lui-même).

### 📊 Diagnostics drift check améliorés

- 🔍 Le log `EnsureUsersDisabledAsync` liste désormais les **usernames** des comptes ré-disabled, plus seulement leurs UUIDs : *"Drift check re-disabled 1 user(s): alice."*
- 🎯 Debug d'une dérive ("Maman ré-active Alice pendant la maintenance") devient instantané sans avoir à matcher l'UUID à un compte.

### ⚙️ GHA opt-in Node.js 24

- 🚀 Workflows CI + release reçoivent `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true` au niveau env.
- 📅 Les runners GHA forceront Node 24 par défaut en **juin 2026** et retireront Node 20 en **septembre 2026** — mieux vaut surfacer toute incompatibilité maintenant tant que le fallback existe.

## 📚 Documentation

- 📖 `docs/webhooks.md` : section `Restarting` ajoutée + champ `NotifyOnRestart` documenté + warning host non-Discord/Slack + note rate-limit global v0.3.3.
- 📚 README : table récapitulative des **9 endpoints REST** avec leurs niveaux d'auth + nouvelle section *Debug flag* qui documente `?md-debug=1`.
